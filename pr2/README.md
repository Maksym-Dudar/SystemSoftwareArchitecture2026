# Практична робота №1

Запустіть Docker-контейнер і поекспериментуйте з максимальним лімітом ресурсів відкритих файлів. Для цього виконайте команди у вказаному порядку:
$ ulimit -n
$ ulimit -aS | grep "open files"
$ ulimit -aH | grep "open files"
$ ulimit -n 3000
$ ulimit -aS | grep "open files"
$ ulimit -aH | grep "open files"
$ ulimit -n 3001
$ ulimit -n 2000
$ ulimit -n
$ ulimit -aS | grep "open files"
$ ulimit -aH | grep "open files"
$ ulimit -n 3000

Як наступне вправу, повторіть перераховані команди з root-правами.

### Опис

Експерименти показали, що стандартний ліміт дескрипторів обмежує кількість одночасно відкритих файлів. Підвищення ліміту (ulimit -n) дозволяє працювати з більшою кількістю файлів. Root-правами можна змінювати ліміт вище стандартного. При зменшенні значення система обмежує процеси і вони не можуть відкривати нові файли понад ліміт.

## Завдання 2
У Docker-контейнері встановіть утиліту perf(1). Поекспериментуйте з досягненням процесом встановленого ліміту.

### Приклад роботи

<img width="916" height="724" alt="Знімок екрана 2026-03-07 о 17 35 57" src="https://github.com/user-attachments/assets/e7450ca3-805e-4c1a-b078-e25a4e13f367" />

### Опис

Perf дозволяє відслідковувати витрати ресурсів процесами. Експеримент показав, що при досягненні встановленого ліміту (CPU, пам’ять) процес не може перевищувати ресурсів, система сигналізує про обмеження. Це демонструє контроль за навантаженням у контейнерах.

## Завдання 3

Напишіть програму, що імітує кидання шестигранного кубика. Імітуйте кидки, результати записуйте у файл, для якого попередньо встановлено обмеження на його максимальний розмір (max file size). Коректно обробіть ситуацію перевищення ліміту.

### Опис

## Завдання 5

Напишіть програму для копіювання одного іменованого файлу в інший. Імена файлів передаються у вигляді аргументів.
Програма має:
перевіряти, чи передано два аргументи, інакше виводити "Program need two arguments";
перевіряти доступність першого файлу для читання, інакше виводити "Cannot open file .... for reading";
перевіряти доступність другого файлу для запису, інакше виводити "Cannot open file .... for writing";
обробляти ситуацію перевищення обмеження на розмір файлу.
